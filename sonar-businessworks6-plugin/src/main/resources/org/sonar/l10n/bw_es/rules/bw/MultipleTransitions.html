Actividades EMPTY deben ser utilizados para unir diferentes transiciones. Por ejemplo, hay multiples actividades de salida de una actividad, y cada una de ellas toma un diferente rumbo en el proceso. En este escenario puedes crear una transicion desde la actividad al final del flujo a una actividad Empyy para volver a crear el flujo único de ejecución en el proceso. Esta regla comprueba si multiples transiciones desde un actividad en paralelo se unen en una actividad Empty.
